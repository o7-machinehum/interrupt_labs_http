It starts as a boring flash drive.

Plug Phantomdrive in and the host sees a regular USB mass storage device. The first section behaves like normal storage. The rest stays hidden until the unlock pattern is written...

Phantomdrive Unlock Flow.

  1. 01

    Mounts normally

    Phantomdrive first appears as a normal USB drive.

  2. 02

    Waits for the trigger

    Create file called unlock.txt containing password:addyourpasswordhere.

  3. 03

    Intercepts the write

    The firmware does not allow the password file write to persist to storage.

  4. 04

    Reveals the rest

    The remaining disk is remounted and encrypted/decrypted in place with AES-256.

Project Details

  • The Hardware,firmware, software, and mechanical files are open source.
  • Easy Updates - The CH569W's firmware can be updated by pressing the reset button, and inserted it into your PC. No programmer required.
  • The KDF,or "key derivation function" is open.
  • We've recorded20MB/s read and 9MB/s write speeds.
  • Designed in KiCad,so don't need to pay thousands to edit the design files
  • UARTis available though TP1, TP2, TP3.

FAQ

Questions and Answers

Why crowdfunding? Why not Kickstarter?

I would like to know how many units to order for the first batch. This simplifies distribution and helps me understand demand. I have use crowdfunding platforms before, they take a cut which raises the price for you.

How far along is the project?

The egineering is finished, I just need to know my volumes to place an order.

Isn't this just VeraCrypt?

No. A normal encrypted volume exposes random-looking encrypted data. PhantomDrive first presents as a normal USB drive, then reveals the protected section only after the unlock trigger.

Is entering a password into a text file insecure?

The firmware watches for the unlock write and does not allow that write to persist to storage.

Why use an SD card?

The current prototype uses SD card storage because eMMC pricing and availability were not attractive for this revision. A future eMMC board spin is still possible.

Is it fully open source?

The project goal is full openness across hardware, firmware, and mechanical files.
